## Codebase Patterns
- **Cart Format:** Always use Cart v2 format with `{ version: 2, items: [], updatedAt: string }`
- **Sanity Images:** Use `coalesce(mainImage.asset->url, image.asset->url)` in GROQ queries
- **Backend Enums:** UPPERCASE only (`USER`, `BUYER`, `GROWER`, `ADMIN`)
- **Auth Tokens:** `setAuthToken()` is client-side only - check for SSR before calling
- **Route Protection:** Use `src/proxy.ts` (Next.js 16 renamed from middleware.ts)
- **API Routing:** Email endpoints route based on `NEXT_PUBLIC_EMAIL_SERVICE_ENV`
- **Test Guards:** Long-running intervals should be disabled in test environment by default. Use opt-in flags: `global.__ENABLE_COOKIE_POLLING_IN_TESTS` and `global.__ENABLE_REALTIME_IN_TESTS`.
- **Hero Carousel Heights:** Use reduced heights for better UX - loading/error: h-[300px] md:h-[350px] lg:h-[400px], content: min-h-[400px] md:min-h-[500px] lg:min-h-[600px]
- **Cookie Mock Pattern:** Global mock in `jest.setupMocks.js` via `global.__mockCookies`. Tests access same instance as source code. For cookies.test.ts that tests real implementation, use `jest.unmock('@/lib/cookies')`.
- **HTTP Status Codes:** Malformed JSON should return 400 (Bad Request), not 500. Client errors = 4xx, Server errors = 5xx.
- **Jest Config:** Use `clearMocks: true`, `resetMocks: false`, `restoreMocks: false` to allow persistent mock setup while clearing call history between tests.
- **AI API Calls:** Server-side AI functions (gemini-client.ts, error-handler.ts) must call external APIs directly, NOT through proxy routes. Proxy routes (/api/ai/*) are for client-side use only.
- **Chatbot UI Tests:** When updating chatbot UI, ensure tests match new aria-labels, text content, and class structures (e.g., `aria-label="Open MASH AI Assistant"`, loading text, wrapper positioning).
- **VPN Network Issues:** When Radmin VPN or similar VPN is active, fetch() may fail in Turbopack/Next.js dev environment. Use native Node.js https module as fallback (gemini-native.ts). Force IPv4 with `family: 4` option.
- **AI Client Retry Logic:** Use custom AbortController-based timeout instead of AbortSignal.timeout() for better compatibility. Implement 2-retry logic with exponential backoff for network resilience.
- **HuggingFace API:** Use OpenAI-compatible chat completions endpoint (`/v1/chat/completions`) with `{model, messages}` format.
- **Test Stability:** All 871 tests pass consistently. Test coverage expanded with Order, Search, and Common component tests.
- **Radix UI Testing:** Radix UI Select/Dialog use portals - test UI presence/structure rather than complex async interactions. Use getAllByText/getAllByRole for multiple elements.
- **Order Type:** Use `Order` from `@/hooks/useSanityOrders` with fields: customerName, customerEmail, customerPhone, shippingAddress (with fullAddress), items with product object.
- **SearchDialog Mock Pattern:** Use callCount-based mockImplementation for Promise.all parallel calls (odd calls = products, even calls = categories).
